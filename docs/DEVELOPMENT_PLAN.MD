# MyChatApp 开发计划

> 项目目标：开发一款轻量级、跨平台（Windows/macOS/Linux）的桌面聊天客户端，用于内部小群体试用。  
> 技术栈：C++17 + Qt6 + CMake + WebSocket  
> 当前阶段：第一阶段 — 基础框架搭建

## 🗓 总体周期安排（预计 8～10 周）

| 阶段 | 时间 | 核心目标 | 交付物 |
|------|------|--------|--------|
| **第一阶段：基础框架搭建** | 第 1–2 周 | 建立可跨平台构建的项目骨架 | 可运行的空白客户端，支持平台编译与打包 |
| **第二阶段：通信核心实现** | 第 3–5 周 | 实现登录、消息收发、本地缓存 | 两人可在内网互发文本消息 |
| **第三阶段：用户体验优化** | 第 6–7 周 | 完善 UI/UX、联系人列表、通知 | 看起来“像一个能用的产品” |
| **第四阶段：内部试用与迭代** | 第 8–10 周 | 邀请种子用户，收集反馈，修复问题 | 稳定可用的 v1.0 内部版本 |

## 🧱 第一阶段详细目标（第 1–2 周）

### ✅ 验收标准
- [ ] Git 仓库初始化完成（含 `.gitignore`, `README.md`）
- [ ] 项目目录结构清晰（`src/`, `resources/`, `docs/` 分离）
- [ ] CMake 支持 Windows / macOS / Linux 构建
- [ ] 启动后显示主窗口（Qt Widgets）
- [ ] 集成 `QSettings` 配置管理与基础日志宏
- [ ] 提供各平台简易打包说明
- [ ] 编写《开发规范》初稿（代码风格、分支策略等）

### 🔧 技术要点
- 使用 Qt6 的 `Widgets` 模块（暂不使用 QML，降低复杂度）
- CMake 最低版本：3.16
- 不依赖第三方网络库（第二阶段再引入）
- 所有资源通过 `.qrc` 或相对路径管理

## ⚠️ 风险与应对

| 风险 | 应对措施 |
|------|--------|
| Qt6 在某些 Linux 发行版安装困难 | 优先使用官方在线安装器或 vcpkg |
| 多人协作时代码风格混乱 | 明确命名规范并写入 `DEVELOPMENT.md` |
