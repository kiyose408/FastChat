# 网络可靠性测试指南

## 测试环境

- 后端: http://localhost:3000
- WebSocket: ws://localhost:3000/websocket

## 测试用例

### 1. WebSocket 自动重连测试

**步骤：**
1. 启动后端服务
2. 启动客户端并登录
3. 停止后端服务 (Ctrl+C)
4. 观察客户端日志，确认开始重连
5. 重启后端服务
6. 观察客户端自动重连成功

**预期结果：**
- 客户端显示 "正在重连..." 日志
- 后端重启后自动连接成功
- 好友在线状态恢复正常

### 2. 心跳机制测试

**步骤：**
1. 启动客户端并登录
2. 保持客户端运行 2 分钟
3. 观察控制台心跳日志

**预期结果：**
- 每 30 秒发送一次 ping
- 收到 pong 响应
- 不会触发重连

### 3. 消息队列测试

**步骤：**
1. 启动后端和客户端，登录成功
2. 停止后端服务
3. 发送一条消息
4. 重启后端服务
5. 观察消息是否自动发送

**预期结果：**
- 断线时消息入队
- 重连后消息自动发送
- 对方收到消息

### 4. HTTP 重试测试

**步骤：**
1. 启动客户端并登录
2. 停止后端服务
3. 点击刷新好友列表
4. 观察重试日志
5. 重启后端服务

**预期结果：**
- 请求失败后自动重试
- 重试间隔递增 (1s, 2s, 4s)
- 服务恢复后请求成功

## 日志关键字

在客户端控制台中搜索以下关键字：

| 关键字 | 说明 |
|--------|------|
| `Heartbeat started` | 心跳启动 |
| `Ping sent` | 发送心跳 |
| `Pong received` | 心跳响应 |
| `Heartbeat timeout` | 心跳超时 |
| `Scheduling reconnect` | 计划重连 |
| `Attempting to reconnect` | 尝试重连 |
| `WebSocket connected` | 连接成功 |
| `Flushing message queue` | 发送队列消息 |
| `Retrying request` | HTTP重试 |

## 常见问题排查

### Q: 重连一直失败

检查：
1. 后端服务是否启动
2. Token 是否过期
3. 网络是否正常

### Q: 心跳超时但未重连

检查：
1. 后端是否正确响应 pong
2. 网络延迟是否过高

### Q: 消息队列未发送

检查：
1. 重连是否成功
2. 消息是否正确入队
